# CODEX 项目日志（整理版）
更新时间：2026-04-07

## 1. 会话目标与范围
- 持续完善管理员端、教师端、学生端的账号与学习流程联动。
- 当前重点：
1. 教师端权限管理（仅本门店学生）
2. 管理员端账号管理与教师端数据一致
3. 登录状态严格依赖数据库（启用状态 + 截止日期）
4. 文案与编码规范稳定执行，避免乱码与语言回退

## 2. 已确认的业务规则
- 教师端账号管理能力与管理员端“字段内容”一致，但教师仅可操作本门店学生。
- `storeName` 不允许被教师修改。
- 教师端支持：
1. 单行编辑、单行保存
2. 单个删除与批量删除（仅本门店学生）
3. “添加学生账号”（弹窗录入，不使用 CSV/Excel）
- 手机号可作为唯一标识；若已存在必须报错，禁止写入。
- 字段强校验：新增/编辑必须完整覆盖关键字段，不允许关键字段缺失（如截止日期为空）。
- 管理员端与教师端使用同一份数据库数据；修改后另一端手动刷新可见。

## 3. 登录与账号状态规则（强制）
- 登录校验严格依赖数据库实时状态。
- 若 `active = false`，登录必须被拒绝，并返回“账号已停用”。
- 若 `expireDate` 到期，登录必须被拒绝，并返回“账号已到期”。
- 禁止仅依赖前端缓存状态判断登录可用性。

## 4. 运行与联调要求
- 每次进入新会话，先执行：
1. 阅读本日志
2. 验证 `run_all` 启动后的端口监听与关键接口可用性
- 若关键服务未启动或端口异常，先修复运行环境，再继续功能开发。

## 5. 文案与编码规范（强制，唯一版本）
- 仓库文本文件统一使用 UTF-8（无 BOM）。
- Windows 终端编辑中文时，必须显式使用 UTF-8 编码写入。
- 平台面向中文用户，前端可见业务文案默认使用中文。
- 除接口字段、协议字段、第三方专有名词外，不得将可见业务文案改为英文。
- 提交前必须检查：按钮、表头、提示、弹窗、校验信息均为中文且无乱码。
- 若发现乱码或英文文案回退：先修复文案与编码问题，再继续开发。

## 6. 教材版本与权限数据规则
- 教材版本、年级范围等权限选项必须来源于数据库查询，不得硬编码或混入杂乱数据。
- 账号管理页面与单词页面使用同一套数据库教材版本来源，保持一致。
- 发现脏数据时，先清理并校验后再发布。

## 7. Git 与 GitHub 每日同步规则（新增）
- 每天开发结束后必须同时完成两件事：
1. 更新 `CODEX_LOG.md`（记录当日改动与关键决策）
2. 提交并推送到 GitHub 远端 `main`
- 标准流程：
1. `git -C d:\zip status`
2. `git -C d:\zip add -A`
3. `git -C d:\zip commit -m "feat: <当日变更摘要>"`
4. `git -C d:\zip push`
- 若推送失败，不得跳过：需先解决认证/网络/冲突后再结束当日工作。

## 8. 提交流程要求
- 修改账号相关功能后，必须完成最小闭环验证：
1. 教师端新增/编辑/删除（含批量）学生账号
2. 管理员端刷新后数据同步可见
3. 关闭启用状态或设置到期后，登录被拒绝
- 未完成验证不得标记为“完成”。
