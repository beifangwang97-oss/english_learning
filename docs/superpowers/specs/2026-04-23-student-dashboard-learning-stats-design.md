# 学生端控制面板词汇轮播与学习统计轮播设计

日期：2026-04-23

## 目标

将学生端控制面板中原先占位性质较强的“已学单词”区域升级为真实功能板块，并充分利用该区域空间，形成两个互补的展示层：

1. 上层词汇轮播
2. 下层学习统计轮播

本次目标不仅是展示数据，还要建立可长期维护的学生学习统计口径。

## 需求收敛结果

本次最终需求确定如下：

### 词汇轮播区

- 展示学生当前年级词库中的一组词汇
- 词汇来源限定为专有名词标签中的：
  - `place`
  - `country_region`
- 展示内容：
  - 英文
  - 中文
- 类似广告位自动轮播
- 鼠标悬浮时暂停
- 鼠标离开后继续轮播
- 同一次登录会话内固定一组词

### 学习统计区

- 统计项不包含单词测试
- 保留以下统计项：
  - 已学单词
  - 已学短语
  - 已练习课文
  - 已完成单词复习
- 每项同时维护两套口径：
  - 总完成
  - 今日完成
- 展示方式采用自动轮播：
  - 一屏显示“总完成”
  - 一屏显示“今日完成”
  - 自动切换
  - 鼠标悬浮暂停
  - 鼠标离开继续

## 为什么需要后端统计表

本次建议新增后端统计表维护学习统计，而不是完全由前端临时汇总。

原因：

1. 当前已经不是单一数字展示，而是多指标统计板块
2. 同时存在“总完成”和“今日完成”两套口径
3. 数据来源分散在多个表中：
   - 单词/短语/课文：学习分组进度
   - 单词复习：复习任务与每日复习会话
4. 若始终靠前端现算，后续会出现：
   - 请求数增长
   - 页面初始化变慢
   - 统计口径分散在前端，难以维护

因此本次推荐建立后端统一统计表与统计接口。

## proper_noun_type 口径确认

根据 `tool/mode3_word_syllable_fill.py`，`proper_noun_type` 的合法值为：

- `person`
- `place`
- `country_region`
- `organization`
- `other`
- `null`

因此本次词汇轮播明确筛选：

- `place`
- `country_region`

## 范围

### 前端

- `front/src/pages/StudentDashboard.tsx`

### 后端

- 推荐在 `backend/test-service` 中新增统计表和统计接口

说明：

- 单词、短语、课文完成事件目前主要来自 `LearningGroupProgress`
- 单词复习完成事件主要来自 `WordReviewDailySession`
- 这些都在测试服务侧更接近学习行为数据

### 本次不做

- 不统计单词测试
- 不做图表化趋势分析
- 不做周/月历史统计页
- 不做排行榜
- 不做老师端查看学生控制面板统计

## 页面结构设计

建议该区域拆成上下两层。

### 上层：词汇轮播卡

#### 展示内容

- 大标题：英文词汇
- 副标题：中文释义
- 小标签：例如 `地名词汇` / `国家地区`

#### 数据来源

- 当前学生的教材版本与年级
- `word` 类型词条
- `proper_noun_type in ('place', 'country_region')`

#### 取词规则

- 先按当前学生 `textbookVersion + grade` 过滤词库
- 仅保留：
  - `type = word`
  - `proper_noun_type = place`
  - `proper_noun_type = country_region`
- 去掉英文为空或中文释义为空的词条
- 以英文词条去重，避免轮播重复
- 从符合条件的词条中随机抽取最多 `10` 个候选词
- 每次轮播只显示 `1` 个词
- 若结果少于 `10` 个，则按实际数量展示
- 若结果为 `1` 个，则不自动切换
- 若结果为 `0` 个，则显示空状态

#### 英文与中文展示字段

- 英文：词条 `word`
- 中文：优先取 `meanings[0].meaning`

#### 行为

- 自动轮播切换
- 鼠标悬浮暂停
- 鼠标离开继续
- 轮播到末尾后循环

#### 固定策略

- 同一次登录会话内固定同一组候选词
- 该固定策略由前端维护，不要求后端保存轮播结果
- 学生本次登录后第一次进入控制面板时随机抽取一组词
- 该组词在本次登录期间保持不变，仅在组内轮播
- 学生退出登录并重新登录后，再重新随机一组词

#### 空状态

- 若没有符合条件词条，显示静态提示：
  - `当前暂无地名或国家地区词汇推荐`

### 下层：学习统计轮播区

#### 轮播屏一：总完成

显示 4 张统计卡：

- 已学单词
- 已学短语
- 已练习课文
- 已完成单词复习

#### 轮播屏二：今日完成

同样显示 4 张统计卡：

- 今日已学单词
- 今日已学短语
- 今日已练习课文
- 今日已完成单词复习

#### 行为

- 自动在两屏之间切换
- 鼠标悬浮暂停
- 鼠标离开继续

## 统计口径定义

### 1. 已学单词

来源：

- `LearningGroupProgress`
- 模块：`vocab`

统计值：

- `total_words_completed`
- `today_words_completed`

口径建议：

- 以完成分组时写入的 `learnedCount` 为准
- 若 `learnedCount` 缺失但分组已完成，则回退使用 `itemTotal`

### 2. 已学短语

来源：

- `LearningGroupProgress`
- 模块：`phrase`

统计值：

- `total_phrases_completed`
- `today_phrases_completed`

口径与单词一致。

### 3. 已练习课文

来源：

- `LearningGroupProgress`
- 模块：`reading`

统计值：

- `total_passages_completed`
- `today_passages_completed`

口径建议：

- 按已完成课文分组数统计
- 每完成一篇课文计 1

### 4. 已完成单词复习

来源：

- `WordReviewDailySession`

统计值：

- `total_review_words_completed`
- `today_review_words_completed`

口径建议：

- 单位按“单词个数”统计，而不是按“复习任务数”
- 每次学生成功提交每日复习结果时，按该次会话实际完成的词数累加

## 后端数据模型设计

建议新增一张学生学习统计表，例如：

- 表名：`student_learning_stats`

字段建议：

- `id`
- `user_id`
- `stats_date`
- `total_words_completed`
- `today_words_completed`
- `total_phrases_completed`
- `today_phrases_completed`
- `total_passages_completed`
- `today_passages_completed`
- `total_review_words_completed`
- `today_review_words_completed`
- `created_at`
- `updated_at`

### 字段设计说明

- `user_id`
  - 关联学生
- `stats_date`
  - 表示“today_*”字段所对应的自然日
  - 用于跨天后将今日统计重置到新日期

## 时间口径

今日统计统一按 `Asia/Shanghai` 计算，即北京时间自然日。

## 统计维护方式

### 推荐方式

采用“事件发生时增量更新统计表”的模式。

即：

- 完成单词组 -> 更新单词统计
- 完成短语组 -> 更新短语统计
- 完成课文组 -> 更新课文统计
- 提交复习日任务 -> 更新复习统计

### 今日统计更新规则

每次更新统计前：

1. 读取该学生统计记录
2. 若 `stats_date` 不是今天
   - 重置所有 `today_*` 字段为 0
   - 将 `stats_date` 改为今天
3. 再执行本次增量累加

## 接口设计

建议新增一个学生控制面板学习统计接口，例如：

- `GET /api/tests/student-learning-stats/{userId}`

返回建议：

- `totals`
  - `words`
  - `phrases`
  - `passages`
  - `reviews`
- `today`
  - `words`
  - `phrases`
  - `passages`
  - `reviews`
- `properNounWords`
  - 轮播词条数组，每项包含：
    - `word`
    - `meaning`
    - `proper_noun_type`

也可以拆成两个接口：

1. 学习统计接口
2. 词汇轮播接口

本次建议优先合并为一个控制面板专用接口，减少前端请求数。

## 前端实现方式

在 `StudentDashboard.tsx` 中新增一个独立板块，包含：

### 词汇轮播状态

- 当前显示索引
- 定时器
- hover 暂停状态

### 统计轮播状态

- 当前显示模式：`totals` / `today`
- 定时切换
- hover 暂停状态

### 渲染结构

- 上层：轮播词汇卡
- 下层：4 张统计卡

## 样式建议

### 词汇轮播卡

- 视觉上更突出
- 强调英文词汇
- 中文做辅助说明
- 类型标签用于区分 `地名` / `国家地区`

### 统计卡

- 延续当前学生端控制面板风格
- 与单元学习卡片风格协调，但不必完全复制
- 推荐使用统一尺寸的小型成果卡

## 风险与取舍

### 风险

- 若不建统计表，前端汇总会越来越重
- 若复习统计按任务数统计，会与你明确提出的“按单词个数”口径不一致
- 若 proper_noun_type 只按单一值匹配，容易漏掉 `place` / `country_region` 的目标数据

### 本次取舍

- 不统计单词测试
- 复习按完成单词数统计
- 词汇轮播只做 `place` 和 `country_region`
- 控制面板以自动轮播为主，不加复杂切换控件

## 验证场景

至少验证以下场景：

1. 有 `place` / `country_region` 词条时，词汇轮播正常显示
2. 无目标词条时，词汇轮播显示空状态
3. 词汇轮播自动切换，hover 时暂停
4. 学习统计区能在“总完成 / 今日完成”两屏间自动切换
5. 学习统计区 hover 时暂停
6. 完成单词组后，单词统计正确累加
7. 完成短语组后，短语统计正确累加
8. 完成课文后，课文统计正确累加
9. 提交单词复习后，复习统计按单词个数正确累加
10. 跨天后，今日统计正确归零并重新累计

## 实现顺序

建议按以下顺序落地：

1. 确定统计表结构与迁移
2. 在学习完成事件中接入统计更新
3. 新增控制面板统计接口
4. 在学生端控制面板中接入词汇轮播
5. 在学生端控制面板中接入统计轮播
6. 验证跨天与 hover 暂停行为

## 结论

本方案将学生端控制面板该区域升级为“词汇推荐 + 学习成果展示”双层结构：

- 上层增强趣味性与记忆点
- 下层沉淀真实学习成果

并通过后端统计表统一维护“总完成 + 今日完成”口径，为后续扩展更多学生成长展示能力提供基础。
