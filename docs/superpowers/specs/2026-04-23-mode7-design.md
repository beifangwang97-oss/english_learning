## Mode7 设计说明

日期：2026-04-23

### 目标

- 恢复 `tool/mode7_exam_extract.py` 为可运行的独立 mode。
- 修复人教版初中同步题中阅读块切分不稳导致的题干、选项、材料串题问题。
- 补齐 `qwen/qwen-2.5-vl-72b-instruct` 模型选项。
- 支持多 API key 并发处理“需要增强”的题目，每个任务只占用一个 key，完成后立即移出任务池。
- 在前端按文件动态展示处理进度，文件可展开折叠，展开后固定高度内部滚动，并实时追加新增题目结果。

### 范围

- 本轮聚焦 mode7。
- `reading_qa` 继续保留为未支持题型，不纳入本轮增强范围。
- API 增强统一使用一套 `base_url + model`，API 列表只维护 key。

### 实现思路

- 先按题型解析试卷和教师稿，再标准化为一题一条 JSONL。
- 阅读理解改为“按材料标签 A-F 切段，再在每段内部解析题目”，避免跨材料串题。
- 对缺失解析、选项不完整、题干异常的题目建立增强任务。
- 使用固定大小线程池并按 key 数量并发提交增强任务，每个 future 绑定单个 key。
- 前端维护按文件分组的实时结果状态，并在任务完成时增量刷新对应文件面板。

### 校验方式

- 用 `tool/exam_data/人教版初中/*.jsonl` 对照检查阅读题的选项完整性。
- 运行 `python -m py_compile tool/mode7_exam_extract.py`。
- 对生成结果统计缺失答案、缺失解析、选项数量异常题目。
